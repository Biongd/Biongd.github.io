使用github部署个人博客站点

发表于 2020-07-07   |   更新于 2021-07-29 | 分类于 git/github-制作图床.博客部署及主题修改.使用教程.远程.

工具：hexo框架

目的：搭建可访问的个人博客

1.node.js下载（官网选择左侧永久版）

下载完成后用到的主要看两项: 终端：node -v看node版本，npm -v看包管理器版本,两个都有就ok 注意：

由于npm包管理器下载镜像源速度很慢，利用npm安装cnpm

为啥要用淘宝的镜像呢？

就是根据国情，很多资源在你的局域网里是访问不了的，淘宝帮你把外网的东西>>下载好了(拷贝了一个一

毛一样的资源整站)，放在它的服务器上，让你能够高速>下载，这就是镜像。npminstall的时候，指定镜像

地址或者直接用淘宝的cnpm即可。

代码:

`npm install -g cnpm --registry-https://registry.npm.taobao.org`

registry：说明镜像源指向淘宝

2.hexo框架的安装：

2.1第一次安装

2.1.1安装：npm install -g hexo

2.1.2初始化项目：hexo init

直接在系统命令提示符中运行报错，改用右键-管理员运行依然报错

直接在git bash here 中运行OK，后续命令都是在git bash中运行

启动：hexo s 即hexo start默认在本地localhost：4000启动，用于本地预览，此时可以看到主题的页面框架

2.1.3新建一篇文章：

hexo n "我的第一篇博客"

即hexo new新建文件，文件路径就在~/source/_posts下

cd f:Biongd/blog/source/_posts

ls -l

查看文件是否成功建立

vi 我的第一篇博客.md

编辑新建文档（markdown格式） 这里也可以通过编辑软件修改（typora）

再次hexo s启动，本地预览文档是否满足要求

cd ../..

返回两级文件夹，回到blog文件夹下

hexo clean

清除缓存文件 db.json 和已生成的静态文件 public

hexo g

即generate，自此本地数据html，css，categories，tags，post等数据收集到public文件夹，后面提交到远端

3.在别的电脑上同步*

重复上述1，2步骤，替换hexo初始化路径下的所有文件（除了.deploy_git和public文件夹，这两个部署到远端才能用到）。

报错1：ERROR Deployer not found: git

解决：重新npm install --save hexo-deployer-git

4.5码云和github的区别

上传到码云需要提交申请gitee page页面才行，通过进入仓库-服务-Gitee pages。每次部署到码云之后都得通过页面来更新才能同步到gitee。详情见Hexo博客部署到码云和Coding

总结

步骤：初始化环境+创建文件+修改配置+推向远端

每次对博客做出修改，如new一个博客，更改博客内容，更换博客主题等，在部署前都需要重新生成hexo clean+hexo g

当然也可以先通过hexo s本地预览

6.执行hexo命令时出现的问题

参考文章：hexo搭建博客：常见错误

  • 一上来命令就执行不了

    问题一

    YAMLException: incomplete explicit mapping pair; a key node is missed; or followed by a non-tabulated empty line at line 1, column 33:

    问题原因

    因为之前刚因为hexo渲染流程图更改了config文件，所以就直接定位了

    解决方案：clean的时候就报错基本就是config文件的格式出了问题！

    （1）出现这种情况，一般都是缺少空格，在:冒号之后要有空格！检查x行y列附近的冒号，其之后是否跟了空格。
    （2）_config.yml文件中所有冒号后面的空格，格式很严格，必须是只有一个，半角。不管是多了还是少了都会报错，这是yml解释器所定义的语法。如果不确定的话，将输入法调整到英文模式，删除所有冒号后面的空格重新输入，不要使用Tab。

    问题二

    hexo: line 12: node: command not found

    问题原因：整理文件夹时修改了node.js的文件夹名称，导致环境变量失效

    解决方案：这一问题基本都是node.js的路径出错或环境变量误删导致，检查一下就好

  • 部署hexo d时出错：

    一般是网络问题导致，重新执行一遍部署步骤就好

  • 文章中的别的问题也很有价值：

    • 1. 执行命令hexo server，提示：Usage: hexo …. 提示找不到该指令
    • 2. 执行：hexo server，启动了服务，在浏览器输入本地地址访问，看到白板和Cannot GET / 几个字
    • 3. localhost:4000 端口被占用，可执行 hexo s -p 5000 修改成5000 端口
    • 5. ValidationError: ‘null’ is not a string!
    • 6. 执行hexo g命令后，报错：incomplete explicit mapping pair; a key node is missed
